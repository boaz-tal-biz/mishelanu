## 2. Multiple Providers Per Request

Right now, the system finds a match and sends one provider. You asked: what if we offer more than one?

**When there are multiple matching providers, who should choose which one to forward — the admin, or the person who made the request?**

For example:
- **Option A:** Admin sees all matches, picks the best one(s) to forward
- **Option B:** Requester receives a shortlist and picks for themselves

## 3. Reporting & Statistics

You mentioned reporting is missing. To build the right reports, it helps to know what you want to track. Some options:

- Number of requests per week/month
- Match success rate (how often a provider says YES)
- Provider response times
- Which categories get the most demand
- Provider popularity / profile visits
- How many providers are live vs pending

**Which of these matter most to you? Anything else you'd want to see at a glance?**

You said only admins should create categories. That's actually how it works today — when a provider picks "Other", they type a suggestion, but it only becomes a real category if admin approves it.

**Do you want to keep that suggestion flow (you still control what gets added), or remove "Other" entirely so providers can only pick from the existing list?**
